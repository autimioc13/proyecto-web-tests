   Purpose: Replace localStorage with database storage
   Index: idx_carts_user_id
   RLS: User accesses only own cart

3. orders
   Columns: id (TEXT PK, format ORD-YYYY-xxxxxxxx), user_id (UUID FK),
            total_price (INT), status (enum), payment_method (enum),
            customer_name, customer_email, created_at, updated_at
   Purpose: Store completed orders
   Indexes: idx_orders_user_id, idx_orders_created_at
   RLS: User sees own, admin sees all

4. order_items
   Columns: id (UUID PK), order_id (TEXT FK), product_id (TEXT),
            product_title, product_price (INT), quantity (INT), created_at
   Purpose: Individual items in orders
   Indexes: idx_order_items_order_id, idx_order_items_product_id
   RLS: User sees items in their orders only

================================================================================
SECURITY IMPLEMENTATION
================================================================================

ROW LEVEL SECURITY (RLS):

Status: ✅ ENABLED on all 9 tables

Key Policies:
- users table: Users read/write only their own profile
- user_roles: Users read own role, admins read all
- carts: Users manage only their cart
- orders: Users see own orders, admins see all
- order_items: Users see items from their orders only
- Sessions, results, activity_logs: User isolation
- Tests: Public read access

Result: Zero-trust architecture. Users cannot access others' data even
with direct API calls or service key compromise.

Additional Security:
- Foreign key constraints enforce referential integrity
- Cascading deletes prevent orphaned records
- UNIQUE constraints (1 cart/role per user)
- Check constraints enforce valid enum values
- Service role key protected in environment variables
